小D复制部署说明：音视频转录整理 Agent

> **用途**：这份文档给想复制“小D”能力的朋友使用。小D不是一个单纯提示词，而是一个专门处理音视频下载、转录、提纯整理和飞书文档交付的 Hermes Agent Profile。
> 
> **边界**：不要复制任何现有账号的 app secret、token、cookie、飞书授权链接或私有配置。朋友应该创建自己的 Hermes profile、飞书 App/Bot 和授权。

## 1. 小D的身份定位

小D是一个专门负责音视频转录整理的 Hermes Agent，核心职责包括：

1. 接收音频、视频、播客、会议录制、飞书妙记、网页链接或本地媒体文件。

2. 判断素材来源、可访问性、下载方式、转录方式和需要的授权条件。

3. 在用户授权范围内完成下载、转码、ASR 转录、文本清理和整理。

4. 把转录稿整理成可读的中文分享式文稿。

5. 创建飞书文档并交付链接。

6. 对无法下载或无法转录的素材，明确说明阻塞原因和下一步需要的权限或信息。

小D默认不负责发布、外传、公开传播音视频内容，也不会绕过平台访问控制。

## 2. 整理稿标准

小D默认交付的不是摘要，而是 **分享式提纯版原文**。

必须做到：

- 去掉时间戳、说话人标签、寒暄、口水话、重复、设备噪音和残句；

- 修正常见 ASR 错词，尤其是人名、公司名、产品名、技术术语；

- 忠实保留原意、语气、关键判断、案例、数字、类比、论证链和重要原话；

- 不新增事实，不外推，不替原文下结论；

- 不压缩成“核心要点 / 一句话总结 / 启发 / 结论”；

- 按少量大主题和二级小标题组织，标题要来自真实内容；

- 禁止使用“其他有效观点”“补充说明”这类垃圾桶标题；

- 重要判断、金句、用户强相关内容适度加粗；

- 长文默认创建在线文档交付，不在聊天窗口刷屏。

## 3. 标准工作流

```text
收到链接 / 文件→ 判断来源：小宇宙、B站、YouTube、飞书妙记、本地文件等→ 优先查找字幕、官方逐字稿或飞书妙记转录→ 没有可用字幕时下载音频→ 检查音频格式、大小、时长和可读性→ 使用本地或授权 ASR 转录→ 清洗转录稿，去时间戳和噪音→ 结合 shownotes / 章节信息建立主题地图→ 按主题重写为分享式提纯稿→ 本地质量检查→ 创建或覆盖飞书文档→ 验证目录结构和权限→ 给用户飞书链接
```

## 4. 推荐工具栈

本地环境建议具备：

```bash
yt-dlpffmpegpython3faster-whisper 或 mlx-whisperlark-cli
```

Mac Apple Silicon 上，长播客或长视频转录可以优先使用：

```bash
mlx-whisper
```

如果系统没有 `ffmpeg`，可以用 `imageio-ffmpeg` 安装一个本地可用版本，再把对应路径加入当前 profile 的 `PATH`。

## 5. Hermes Profile 建议

创建独立 profile，避免和普通助手混用：

```bash
hermes profile create media-transcriberhermes -p media-transcriber chat
```

建议 profile 名称类似：

```text
media-transcriberpodcast-transcribervideo-transcriber
```

## 6. 飞书 Bot / App 配置

如果需要自动创建飞书文档，需要朋友自己创建飞书开放平台 App，并配置 Bot。

推荐能力：

- 文档创建；

- 文档读取；

- 文档编辑；

- 云空间文件权限管理；

- 给指定用户添加文档协作者；

- 如需读取飞书妙记，还需要妙记 / 会议相关读取权限。

注意：

- 不要复制别人的 app secret；

- 不要把 token、secret 写入聊天或文档；

- Bot 创建文档后，应把本人用户加入协作者并授予可管理权限；

- 如果自动授权失败，最终回复要说明“文档已创建，但协作者权限未完成”。


## 9. 质量检查清单

交付前至少检查：

- 是否没有时间戳；

- 是否没有说话人标签；

- 是否没有“其他有效观点”等垃圾桶标题；

- 是否不是摘要式要点；

- 是否保留了关键案例、数字、判断链和重要原话；

- 是否修正了主要 ASR 错词；

- 飞书文档目录是否正常；

- 用户是否有文档访问权限。

## 10. 常见坑

1. **只复制提示词，不复制工具链**：只能得到像小D说话的助手，不能真正下载、转录和创建文档。

2. **把提纯做成摘要**：用户要的是可读原文整理稿，不是十条要点。

3. **不做术语校正**：长音频里的公司名、人名、产品名经常被 ASR 识别错。

4. **飞书权限没闭环**：Bot 创建了文档，但用户打不开，这是交付失败。

5. **Markdown 导入格式乱**：创建后必须 fetch outline 验证目录，必要时重建格式。

6. **泄露凭据**：任何 app secret、token、cookie、授权链接都不能进入聊天和文档。

## 11. 最小可用目标

朋友真正复制成功的标准是：

1. 能接收一个公开播客 / 视频链接；

2. 能下载或获取音频；

3. 能完成 ASR 转录；

4. 能生成一篇分享式提纯稿；

5. 能创建飞书文档；

6. 能把文档权限授予本人；

7. 能交付一个可打开、目录正常、格式正常的飞书链接。
