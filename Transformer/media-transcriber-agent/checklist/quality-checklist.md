# 质量检查清单

> 交付前必查的 9 项标准（参考小D 第 9 节）
> 每一项必须通过，否则视为交付失败。

## 检查清单（9 项）

### 1. 是否没有时间戳
- [ ] 检查全文，确认没有 `00:00:00`、`[00:00]`、`-->` 等 SRT/VTT 时间戳格式残留
- ASR 原始稿会带时间戳，提纯后必须全部清除

### 2. 是否没有说话人标签
- [ ] 检查全文，确认没有 `说话人 1:`、`Speaker A:`、`【张三】` 等说话人标签残留
- 多人对话场景尤其要注意

### 3. 是否没有"其他有效观点"等垃圾桶标题
- [ ] 检查所有小标题，确认没有以下无信息量标题：
  - "其他有效观点"
  - "补充说明"
  - "其他内容"
  - "总结"（除非确实是对全文的总结章节）
  - "补充"
- 标题必须来自真实内容，具体且有意义

### 4. 是否不是摘要式要点
- [ ] 检查全文结构，确认不是"核心要点 / 一句话总结 / 启发 / 结论"形式
- 应该是"分享式提纯版原文"，保留原意和完整论证链
- 不应该出现"要点 1、要点 2..."这种压缩式结构

### 5. 是否保留了关键案例、数字、判断链和重要原话
- [ ] 检查关键案例是否完整（背景+过程+结论）
- [ ] 检查关键数字是否准确（金额、比例、日期、数量）
- [ ] 检查判断链条是否完整（前提→推理→结论）
- [ ] 检查重要原话是否保留（金句、标志性言论）

### 6. 是否修正了主要 ASR 错词
- [ ] 检查人名（如"张三"而非"张山"）
- [ ] 检查公司名（如"字节跳动"而非"字节挑动"）
- [ ] 检查产品名（如"ChatGPT"而非"拆 GPT"）
- [ ] 检查技术术语（如"Transformer"而非"转换器"）
- 如果原文有 shownotes/简介，对照修正

### 7. 飞书文档目录是否正常（Phase 2+）
- [ ] 文档大纲层级正确（一级标题 → 二级标题）
- [ ] 目录可正常展开/折叠
- [ ] 没有 Markdown 格式错误导致目录断裂
- Phase 1（本地 Markdown）跳过此项

### 8. 用户是否有文档访问权限（Phase 2+）
- [ ] 本人已添加为协作者
- [ ] 权限为"可管理"（可编辑）
- [ ] 用本人账号实际打开链接验证
- Phase 1（本地文件）跳过此项

### 9. 文稿格式是否规范
- [ ] 标题层级正确（# 一级，## 二级，### 三级）
- [ ] 重要内容已加粗（`**加粗**`）
- [ ] 段落间距合理（段落间空行）
- [ ] 没有 Markdown 语法错误（未闭合的 `**`、`[]` 等）

## 常见质量问题与修复

| 问题 | 原因 | 修复方式 |
|---|---|---|
| 时间戳残留 | 提纯 Prompt 未强调清除时间戳 | 在 refine.py 的系统提示词中强化 |
| 说话人标签残留 | ASR 输出带 diarization 标签 | 在提纯前预处理去除 |
| 垃圾桶标题 | LLM 偷懒，未提炼小标题 | 在 Prompt 中给出正反示例 |
| 变成摘要 | LLM 默认行为是总结 | 在 Prompt 中明确"不是摘要，保留原意" |
| ASR 错词未修 | LLM 缺乏领域知识 | 提供 shownotes/简介作为参考 |
| 飞书目录乱 | Markdown 格式不规范 | 用 lark-cli fetch outline 验证 |
| 文档打不开 | 协作者权限未授权 | 用 lark-drive 手动添加协作者 |
